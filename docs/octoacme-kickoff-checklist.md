# OctoAcme — Project Kickoff Checklist

Use this checklist at the start of every new project or major initiative during the Initiation and Planning phases.

> Related: [Roles & Personas](./octoacme-roles-and-personas.md) · [Roles Interaction Cheatsheet](./octoacme-roles-interaction-cheatsheet.md) · [Project Initiation Guide](./octoacme-project-initiation.md)

---

## Pre-Kickoff (before the meeting)

- [ ] Project One-pager drafted and shared at least 48 hours in advance
- [ ] Stakeholder list confirmed (Sponsor, Product Manager, Project Manager, key Developers, Business Analyst, UX Designer if applicable)
- [ ] Meeting invite sent with agenda and pre-read materials
- [ ] Confirm all required roles are covered — see [Roles Interaction Cheatsheet](./octoacme-roles-interaction-cheatsheet.md)
- [ ] Initial risk list drafted (even if incomplete)
- [ ] High-level timeline and milestones drafted for discussion

---

## During Kickoff

- [ ] Review and agree on problem statement and success metrics
- [ ] Walk through stakeholder list; identify gaps in representation
- [ ] Confirm team roles and responsibilities (use [Roles & Personas](./octoacme-roles-and-personas.md) as reference)
- [ ] Discuss and agree on Definition of Done
- [ ] Review initial risk list; assign owners to top risks
- [ ] Agree on communication cadence (standup frequency, weekly syncs, demos)
- [ ] Confirm tooling: project board, repo, comms channel
- [ ] Identify known dependencies and integration points
- [ ] Agree on decision-making authority (who has final say on scope, design, release)

---

## Post-Kickoff (within 48 hours)

- [ ] Kickoff notes and decisions documented and shared
- [ ] Project board or backlog skeleton created
- [ ] Risk register initialized in the repo
- [ ] Communication plan sent to all stakeholders
- [ ] Confirm sprint/iteration start date
- [ ] Schedule recurring ceremonies (standup, planning, review, retro)
- [ ] Move project to Planning phase — see [Project Planning](./octoacme-project-planning.md)

---

## Roles Present at Kickoff

| Role | Required? | Notes |
|---|---|---|
| Project Manager | Yes | Facilitates kickoff; owns follow-ups |
| Product Manager | Yes | Presents vision and success metrics |
| Developer (lead or representative) | Yes | Provides technical input and feasibility |
| Business Analyst | Recommended | Captures requirements and decisions |
| Scrum Master / Delivery Facilitator | Recommended | Confirms ceremony structure |
| UX Designer | If UI work is in scope | Flags design dependencies |
| Support Engineer | If near release | Plans support readiness |
| Sponsor / Stakeholder | Yes | Confirms priority and funding |
